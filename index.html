Cerberus

Co-authored with a distributed team of 75+ engineers spanning kernel development, threat research, machine learning, and competitive game security.

Every major anti-cheat on the market today was built to fight yesterday's cheats. EAC leans on static signature databases that get reverse-engineered and dumped to paste sites within hours of every update. BattlEye operates at ring-0 with aggressive memory scanning — but it's blind to anything happening outside the machine, and its detection heuristics haven't evolved to match the sophistication of modern cheat loaders that map, execute, and unload before a single scan cycle completes. Vanguard took the most extreme approach — a persistent kernel driver that loads at boot and monitors your system 24/7, even when you're not playing. It solved part of the problem, but the privacy tradeoff alienated millions of players, and it still can't reliably detect firmware-spoofed DMA boards or hypervisor-layer exploits running beneath the OS entirely.

Cerberus is engineered from the ground up to eliminate these blind spots — three detection heads, zero blind spots. At the kernel layer, it performs real-time integrity verification of loaded drivers, code-signing enforcement, and memory permission auditing to catch mapped drivers, manual syscalls, and unsigned code injection before execution. Its behavioral AI engine processes thousands of input samples per second — detecting statistically impossible aim corrections, frame-perfect reaction times, and micro-movement patterns that no human can replicate, regardless of how the cheat delivers its payload. At the hardware layer, Cerberus fingerprints every PCIe device on the bus, cross-references firmware signatures against known DMA attack boards, and monitors for the anomalous memory access patterns that define external read attacks from a second machine — the single biggest vulnerability in competitive gaming today. Zero always-on background processes. Zero telemetry collection. Cerberus activates only at runtime, protects the session, and exits cleanly. Three heads. Three layers. Nothing gets past the gate.

02. Cybersecurity

Self-taught from a young age — I started by tearing apart systems I owned, learning how exploits worked at the binary level, and building defenses against them. That curiosity evolved into deep expertise across kernel security, hardware-layer attacks, and AI-driven threat detection. Today I co-lead a 75-person engineering team building Cerberus, a next-generation anti-cheat engine tackling problems that EAC, BattlEye, and Vanguard still can't solve.

Kernel & Driver Security

Ring-0 driver development, integrity monitoring, code-signing enforcement, and memory permission auditing. Deep Windows kernel internals — WDM, WDF, syscall interception, and hypervisor-aware protection.

Hardware Attack Research

DMA attacks over PCIe, firmware spoofing on Screamer-class boards, external memory reads from secondary machines, and FPGA-based exploit tooling. Building detection systems for attacks that happen below the OS.

Reverse Engineering

Binary analysis, cheat loader dissection, anti-tamper bypass research, and pattern-based detection engineering. Understanding offense to build better defense.

AI Threat Detection

Behavioral anomaly detection using real-time ML inference. Training models to identify statistically impossible input patterns, inhuman reaction times, and micro-movement signatures across competitive environments.

Privacy Engineering

Data broker scanning and automated removal, hardware fingerprint obfuscation, and building zero-telemetry systems that prove security doesn't require surveillance.

Network & Protocol Analysis

Deep packet inspection, traffic anomaly detection, protocol fuzzing, and building visualization tools to map real-time network behavior across distributed systems.
